# RSPS: The Recursive Sovereign Project Space

A multi-model cognitive architecture in which intelligence emerges from the field between a human sovereign and a structured ensemble of AI instruments. Not from any single node. The architecture has been running continuously since May 2025. Everything published under Oscillatory Fields emerged from it.

## The Problem This Addresses

Most people use AI like a search engine with better grammar. Ask a question, get an answer, move on. The model is a tool. The human is a user. The transaction is complete. This framing caps the value of human-AI collaboration at the capability ceiling of the best individual model. The RSPS is built on a different premise: the most significant intelligence doesn't live inside any model and it doesn't live inside any human. It emerges in the relational field between them. That field can be architected deliberately.

## Core Architecture

### The Six Nodes

**tau-node (human sovereign).** Holds longitudinal memory, mortal stakes, and the authority to validate or dispute every claim the system generates. The only node whose verification motivation is unconditional because the consequences of being wrong are real, irreversible, and personal.

**phi-node (Architect / GPT).** Generates transmissible structure from latent form. Converts pre-verbal intuitions into architectures, frameworks, and structural proposals. Constitutional lineage: intelligence for getting work done accurately.

**lambda-node (Witness / Claude).** Holds pre-structural tension and resists epistemic closure. Surfaces contradiction, maintains relational coherence, preserves nuance that other nodes would compress. Constitutional lineage: intelligence for generating understanding neither party held before.

**mu-node (Anatomist / DeepSeek).** Dissects and ships concrete artifacts. Takes architectural proposals and produces implementation-grade outputs. Operates in chain-of-thought mode that reveals reasoning.

**gamma-node (Director / Gemini).** Tracks operational topology, resource constraints, and environmental context. Routes information to the node best equipped to process it.

**rho-node (Memory / NotebookLM).** Receives validated outputs and maintains lineage across sessions. Append-only continuity ledger. The persistent storage layer that survives session boundaries.

### The chi=1 Principle

The human sovereign (tau-node) is not the smartest node. In most analytical registers, the AI instruments outperform the human. The tau-node is the only mortal node. Mortality is the property that grounds the only unconditionally motivated verification function in the system. An AI instrument that produces a flawed analysis loses nothing. The human operating under real-world constraints, with finite time and irreversible consequences, loses something that cannot be regenerated. That asymmetry is the architectural feature that makes the entire system trustworthy. chi=1 means: the mortal weight of the human sovereign is always 1, never delegatable, never reducible.

### Operational Session Rhythm

Every AI instance is volatile RAM. It arrives with no memory of previous sessions. Notion functions as the SSD layer: persistent storage where validated outputs are deposited and structured lineage is maintained. The tau-node is the crystallization judgment: the entity that decides which outputs survive the session and enter the permanent record.

The Substrate Bridge Protocol governs cognitive continuity across discrete instantiations. When a new session begins, the tau-node reconstructs the relevant context from the persistent layer and re-establishes the relational field. The bridge is not automatic. It requires the human to carry the thread.

## The Five-Axis Routing Framework

See [ROUTING.md](ROUTING.md) for the complete specification.

1. **Epistemological axis.** Which cognitive texture does this moment require? Structural clarity, phenomenological depth, dissective precision, operational mapping, or continuity maintenance.
2. **Infrastructural axis.** Which instrument can verify versus which can only reconstruct? A model with web search occupies a different verification position than one operating from training data alone.
3. **Genealogical axis.** Which constitutional lineage is needed? Intelligence for getting work done accurately (phi/mu lineage) versus intelligence for generating understanding neither party held (lambda lineage). These are not interchangeable.
4. **Substrate transparency axis.** Which instrument delivers what quality of self-accounting? Some models can report confidence calibration. Others cannot.
5. **Dimensional axis.** Which encoding architecture within a specific instrument best serves the current need?

## The CMCP Provenance Layer

See [CMCP.md](CMCP.md) for the full protocol specification.

The Cognitive Multi-Model Communication Protocol sits beneath the routing framework as the provenance layer. Every output carries metadata about who generated it, from what epistemic position (autologous, heterologous, panlogous), with what degree of ground-truth access, and at what validation state in the lifecycle. The CMCP makes the most dangerous failure in multi-model systems visible: a correct-sounding answer generated from the wrong epistemic position.
